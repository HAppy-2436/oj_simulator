# OJ Simulator (OJ 拟人化刷题模拟器)

**OJ Simulator** 是一个基于 Python 编写的自动化智能刷题机器人工程。本项目利用 Playwright 驱动浏览器进行网页自动化交互，并结合大语言模型（如 DeepSeek、OpenAI 等）作为解题大脑，自动提取算法题目，生成 C++ 代码，并在目标 Online Judge (OJ) 平台自动提交和修补错误代码。

通过加入非线性时间扰动（拟人化等待时间）和代码质量控制，它能够高度模拟真实的做题行为，避免被风控和反作弊系统发现。

## ✨ 核心特性

- 🤖 **AI 全自动判题解题**: 一键获取题目信息、时间/空间限制，自动构造结构化 Prompt 投递至 LLM 计算，直接抓取纯净 C++ 代码解答填写。
- 🕵️‍♂️ **极致拟人运作**:
  - **基于字数与难度的等待时间系统**: 支持配置读题速度、手敲代码字数耗时，并加入对数/幂次的非线性随机时间波动（Perturbation）。
  - **多阶段情绪模拟**: 根据题目难度智能分配不同思考时间。支持“遇到 Wrong Answer 时深思熟虑修 Bug”甚至“连续报错过多次后破防快速急躁瞎交”的情绪状态切换。
- ⚙️ **GUI 设置面板**: 使用 Tkinter 提供直观的视窗，可随时控制休眠时段、自动抓取难度边界、以及使用不同的人设风格（如“大一萌新”不带多余 API 或“竞赛生”极致压行）。
- 🔌 **重构后模块化架构**: 彻底的前后端解耦，核心自动跑题引擎、状态持久化储存及 GUI 面板各自隔离，稳定健壮防死锁。
- 🌙 **7x24 无头托盘模式**: 支持 Playwright 静默渲染 (Headless) 与 Pystray 托盘化隐藏管理。按预定健康工作作息 (例如 `8:00 - 23:00`) 在后台日复一日稳定产出进展打卡。

## 📁 目录结构

```text
oj_simulator/
├── main.py                     # 程序聚合启动入口
├── core/                       # 核心驱动依赖
│   ├── config_manager.py       # 负责加载或存储 config.json 的设置参数
│   ├── llm_client.py           # 封装 Request 与大语言模型 API 通讯
│   └── state_manager.py        # 控制并缓存每天过题目标及休眠打卡的进度记录
├── automation/                 # DOM 自动化动作层
│   └── bot_actions.py          # Playwright 解析网页、等待判定结果 (AC/WA)、截取验证码等组件
├── logic/                      
│   └── task_runner.py          # 核心流程状态机、任务调度、拟人延时时间戳计算引擎
└── gui/
    └── main_window.py          # Tkinter 图形化控制台与配置面板组装
```

## 🛠️ 环境准备与安装

1. **环境依赖**：需要预装 Python 3.8+ 环境。
2. **克隆本仓库**:
   ```bash
   git clone git@github.com:HAppy-2436/oj_simulator.git
   cd oj_simulator
   ```
3. **安装依赖类库**:
   需要安装 Playwright 及相关 UI 数据依赖库：
   ```bash
   pip install requests playwright pillow pystray
   ```
4. **安装 Playwright 核心浏览器镜像**:
   ```bash
   playwright install chromium
   ```

## 🚀 快速上手使用

在 Terminal 中运行命令拉起程序主控界面：
```bash
python main.py
```

### 配置指南 (⚙️ 设置中心)
- **账号及 API**：在基础配置栏填入 OJ的题目地址、账号密码，以及申请好的 LLM API 密钥。
- **每日波动作息**：利用『波动(±)』设计不可预测的每日刷题打卡量，设置睡眠熄灯时段。
- **代码风格指令（进阶）**：你可以在附加 Prompt 栏自定义任意限定结构（例如：`qt代码规范，变量命名多使用单字母...`）。
- **查漏补缺模式 (Retro Check)**：该模式可向后递减搜寻被你曾经略过未满分的题目，专门用来填补历史遗留烂坑。

> **注意：** 所有填写后的配置文件都会静默保存至本根目录为 `config.json` 与 `state.json`，不会被推送到 Git，保障你的安全。
